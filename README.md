# 📈 기술적 퀀트 전략 백테스트 및 매수/매도 신호 웹앱 (MVP)

본 프로젝트는 미국 주식(US) 및 한국 주식(KR) 시장 종목을 대상으로 다양한 기술적 지표 기반 퀀트 트레이딩 알고리즘을 백테스트하고, 파라미터를 최적화하며, 실시간 매수/매도 타점 및 포지션 비중을 계산하여 시각화해주는 **무료 Streamlit 웹 애플리케이션**입니다.

---

## 🚀 주요 기능 요약
1. **실시간 글로벌 시장 상태 필터**: 주요 지수(SPY, QQQ, KOSPI, KOSDAQ)의 장단기 추세를 바탕으로 시장의 Risk-On/Off 국면을 자동 감지합니다.
2. **기술적 지표 계산**: MA(5~200), EMA(20, 50), RSI, ATR, 볼린저 밴드, Donchian Channel, 기간별 모멘텀 등 퀀트에 핵심적인 지표들을 로컬 메모리에서 신속히 계산합니다.
3. **7대 표준 매매 알고리즘 백테스트**:
   - 전략 A: 추세 모멘텀 눌림목 전략
   - 전략 B: 20일 신고가 돌파 전략
   - 전략 C: 터틀 스타일 55일 돌파 전략
   - 전략 D: 듀얼 모멘텀 전략
   - 전략 E: 변동성 돌파 전략 (K값 최적화)
   - 전략 F: 상승장 RSI 평균회귀 전략
   - 전략 G: 이동평균 크로스 전략
4. **학습(Train) / 검증(Test) 분할 최적화**: 과거 데이터를 70:30으로 나누어 학습 구간에서 최적화한 파라미터가 검증(전방 시뮬레이션) 구간에서 과최적화(Overfitting)되지 않는지 검사하고 경고 메시지를 보여줍니다.
5. **종목 스캐너**: 전체 유니버스 종목들의 기술적 스코어를 가중 합산(0~100점)하여 BUY/WATCH/SELL 관심 종목 리스트와 매수 타겟가, 손절가, 목표가를 도출하고 CSV 다운로드를 지원합니다.
6. **상세 신호 & 포지션 사이징**: 1회 매매당 최대 자산 리스크 노출률(기본 1%) 및 단일 종목 한도(기본 20%)를 고려하여 구체적인 주문 주식 수량과 비중을 제안합니다.
7. **포트폴리오 시뮬레이터**: 리밸런싱 주기에 맞춰 다중 종목 포트폴리오를 분산 운용했을 때의 성과 그래프와 종목별 자산 기여도(Performance Contribution)를 추적합니다.

## 🛠️ 설치 및 로컬 실행 방법

### 1. 사전 요구 사항
- Python 3.11 이상이 로컬 시스템에 설치되어 있어야 합니다.

### 2. 패키지 설치
프로젝트 루트 폴더로 이동한 후, `requirements.txt`에 기록된 의존 패키지들을 설치합니다.
```bash
pip install -r requirements.txt
```

### 3. 로컬 서버 실행
Streamlit 명령어를 통해 웹 브라우저로 애플리케이션을 구동합니다.
```bash
streamlit run app.py
```
실행 후 터미널에 나타나는 로컬 URL(예: `http://localhost:8501`)을 클릭하여 접속합니다.

## 🌐 클라우드 배포 가이드 (무료)

### 1. Streamlit Community Cloud 배포 방법
1. 본 프로젝트 코드를 본인의 **GitHub 저장소(Repository)**에 업로드(Push)합니다.
2. [Streamlit Share](https://share.streamlit.io/)에 접속하여 로그인합니다.
3. **[New app]** 버튼을 클릭한 뒤, 본인의 GitHub 저장소, 브랜치(main), 메인 파일 경로(`app.py`)를 선택합니다.
4. **[Deploy!]** 버튼을 누르면 무료로 배포가 완료되며 상시 가동되는 URL을 획득할 수 있습니다.

### 2. Hugging Face Spaces 배포 방법
1. [Hugging Face](https://huggingface.co/) 계정을 만들고 로그인합니다.
2. **[Spaces]** 탭에서 **[Create new Space]** 버튼을 누릅니다.
3. SDK로 **Streamlit**을 선택하고, Space 하드웨어는 기본 무료 사양(CPU basic)을 고릅니다.
4. 생성된 저장소의 가이드라인에 따라 Git Clone을 하거나 브라우저에서 직접 코드를 업로드합니다. (`app.py`, `requirements.txt`, `src/` 폴더 등 전체 업로드 필요)
5. 업로드가 완료되면 자동으로 빌드(Build) 및 배포가 진행됩니다.

## 📊 데이터 소스 및 API 한계
- **미국 시장**: `yfinance` 라이브러리를 통해 Yahoo Finance에서 주가 일봉을 무료로 다운로드합니다.
- **한국 시장**: `pykrx`와 `FinanceDataReader` 라이브러리를 통해 한국거래소(KRX)와 네이버 금융 포털 등에서 데이터를 교차 수집합니다.
- **캐싱 메커니즘**: 무료 데이터 소스의 호출 빈도 제한(Rate Limit)을 준수하고 로딩 지연을 해결하기 위해 `data/cache/` 디렉토리에 **SQLite3** 기반 로컬 DB 캐시를 기본 적용하였습니다. 동일한 종목에 대한 반복 스캔 시에는 네트워크 호출을 건너뛰고 캐시된 데이터를 즉시 불러옵니다. 캐시 갱신이 필요할 경우 메인 화면 사이드바의 `[로컬 캐시 초기화]` 버튼을 이용해 리프레시할 수 있습니다.

## ⚠️ 백테스트 및 투자 유의사항
- **Look-ahead Bias 금지**: 당일 마감된 데이터를 기준(전일 종가 등)으로 매수 조건이 일치했는지 판단하고, 실제 주문 진입/청산 거래는 다음 영업일 시가 또는 종가에 이루어지도록 룩어헤드 편향을 완벽히 방지하여 백테스트의 정확성을 확보하였습니다.
- **거래 비용 반영**: 시장 평균 수수료와 슬리피지(체결 오차)를 왕복 비용 파라미터(미국 0.20%, 한국 0.35% 기본값)로 모사하여 실전 투자와 유사한 빡빡한 시뮬레이션 환경을 적용했습니다.
- **과최적화 주의**: 특정 기간의 특정 종목에 맞게 이동평균선 일수나 RSI 기준선 파라미터를 과도하게 조절하면, 과거 성과는 환상적이어도 미래 실전 투자에서는 반드시 손실로 이어집니다. 항상 **전략최적화** 페이지의 학습(Train) vs 검증(Test) 성과 비교 격차 및 경고 태그를 확인하십시오.
